Electoral Vote Simulator Documentation

Project Overview

The Electoral Vote Simulator is a Python-based application that simulates a voting process between two candidates through an engaging graphical user interface (GUI). Users can cast their votes for either Candidate 1 or Candidate 2, and the program will keep track of the votes. When the voting ends, the program displays the final results and exports them to a CSV file for record-keeping.

Purpose

This project demonstrates the use of Python for creating interactive applications with a focus on GUI development, input handling, and data export capabilities. It serves as an educational tool for understanding basic to intermediate Python concepts and is also suitable as a portfolio piece for showcasing programming skills.

Key Features

1. Interactive GUI

Engaging User Interface: The program features a user-friendly graphical interface that guides users through the voting process, making it accessible even for those with minimal technical experience.

Simple Navigation: Users can easily start the simulation, enter candidate names, cast votes, and end the voting process with clear on-screen instructions.

2. Voting Simulation

Real-Time Feedback: Users cast votes in real-time, with the program immediately updating and displaying the current vote counts for each candidate.

Final Vote Count: At the end of the voting process, the program displays the total votes for each candidate and the overall number of votes cast.

3. Electoral College Simulation (If Applicable)

State-Based Voting: The program can simulate an electoral college system where votes are tallied based on state-level results, providing a more complex voting scenario (if this feature is implemented).

4. CSV Export

Export Functionality: The final voting results can be exported to a CSV file, making it easy to store and analyze the results after the simulation.

Customizable File Naming: Users can specify the file name for the exported CSV file, allowing for organized data management.
